🗺️ WHAT THIS APP DOES
STLCityRoute is a mobile route planner for City of St. Louis employees who visit multiple addresses in a single shift. Build a stop list, optimize the order to minimize driving, track completions, and keep reference materials in your pocket.

Works offline once loaded. No login. All data stays on your device.
📋 STOPS TAB
Single stop: Type an address — live suggestions appear after 3 characters. Tap a suggestion to add it verified with GPS coordinates. If no suggestion appears, tap ADD STOP to add unverified (⚠).

Batch import:
📷 PHOTO — Requires Anthropic API key in ⚙️ Settings. AI reads the photo and extracts addresses. Without a key, on-screen instructions guide you to use Google Lens (Android) or iOS Live Text to copy text from the photo, then paste it.
📋 PASTE — Paste a list (one address per line), tap EXTRACT ADDRESSES.
📊 SPREADSHEET — Upload a CSV or Excel file.
🎤 VOICE — Tap mic, speak addresses one at a time, tap ⏹ when done, tap ADD ALL.

Duplicate detection: "Blvd" and "Boulevard" are treated as identical. If a duplicate is found you can combine or keep both.

Stop actions: 🧭 NAV opens turn-by-turn navigation. 🏠 PROP copies the address and opens City property search. ✓ CLOSE marks it done.
🛣️ ROUTE TAB
⚡ Optimize: Pings your GPS fresh every time, geocodes any stops missing coordinates, then builds the most efficient route using a nearest-neighbor algorithm followed by a 2-opt improvement pass. After optimizing, the map updates automatically with your GPS position — no need to tap GPS again.

How the algorithm works: Starting from your GPS location (or pinned stop), it always drives to the closest unvisited stop next. A 2-opt pass then checks every pair of stops and keeps any swap that reduces total distance. This eliminates most backtracking and cross-paths in under a second for typical field lists of 10–30 stops.

📌 Pin button: Tap to lock a stop as position 1 for the next optimize. The algorithm starts from that stop and works outward by distance. Tap again to unpin. Use this when you know which end of your route to start from — for example, pinning the southernmost stop before optimizing will build the route heading north.

▲▼ buttons: Move any stop up or down one position. Useful for minor adjustments after optimizing. The map refreshes automatically after each move.

↺ RESET: Clears any optimized or manually adjusted ordering and restores stops to the original sequence you added them. Use this if you want to start fresh before re-optimizing.
🗺️ MAP TAB
Shows all pending stops as numbered colored circles connected by a blue route line. Completed stops are not shown. Tap any circle to see the stop address and access NAV, PROP, MOVE (position picker), and ✓ CLOSE.

Auto-refresh: Any time you add, close, or delete a stop from any tab, the map is flagged stale. Switching to the map tab triggers an automatic refresh. If you return within 5 minutes the cached GPS position is reused — no extra ping needed.

📡 GPS button: Pings your current location AND refreshes the full map in one tap.

MOVE button (in popup): Opens a position picker grid — tap any number to move the selected stop to that position in the route.

Stop colors: Red = Service Request · Blue = Inspection · Orange = other types · Blue dot = your location.
🪪 INFO TAB
ID Card: Tap to view full-screen. Upload photo and set your name/title via ⚙️ Customize.
Questions & Follow-Up: Department contacts and CSB QR code.
Roles & Authority: Role description, enforcement reference, and ordinance text — all editable.
Reference Documents: Store photos of permits, maps, or procedure sheets.
📞 CSB & 🏠 PROP
📞 CSB: Opens Citizens Service Bureau navigator. Select category, copy description, then CALL 311 or REPORT ONLINE.

🏠 PROP: Copies the street address to clipboard and opens the City property search. Paste the address to look up owner, parcel, ward, and other property data.
💡 TIPS
• Start your day: Import your list → ⚡ Optimize → check map → go.
• GPS not working: GPS requires HTTPS. Open the app from your github.io URL and tap ⚙️ → Add to Home Screen. GPS works every time after that.
• Offline use: Once installed as a PWA, the app works without cell service. Address verification and map tiles need a connection.
